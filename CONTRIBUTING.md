# プロジェクト構造 (Project Structure)

このドキュメントは、famous-gameリポジトリの構造と拡張方法について説明します。

## ディレクトリ構造

```
famous-game/
├── python/                     # Python実装
│   └── tic-tac-toe/           # 三目並べ
│       ├── tic_tac_toe.py     # メインゲームロジック
│       ├── test_tic_tac_toe.py # テスト
│       └── README.md          # 実装ドキュメント
├── javascript/                 # JavaScript実装
│   └── tic-tac-toe/           # 三目並べ
│       ├── tic_tac_toe.js     # メインゲームロジック
│       ├── test_tic_tac_toe.js # テスト
│       └── README.md          # 実装ドキュメント
├── .gitignore                 # Git除外設定
├── README.md                  # メインドキュメント
└── LICENSE                    # ライセンス

```

## 新しいゲームを追加する方法

### 1. ディレクトリを作成
```bash
mkdir -p <language>/<game-name>
```

### 2. ゲーム実装ファイルを作成
- メインゲームロジック
- テストファイル
- README.md (実行方法、ルール説明など)

### 3. 実装のガイドライン
- **コメント**: 日本語または英語でわかりやすく
- **変数名**: 英語で記述（国際的な標準に従う）
- **ドキュメント**: 日本語と英語の両方を含める
- **テスト**: 基本的なゲームロジックをカバー

### 4. メインREADMEを更新
新しいゲームを追加したら、ルートの`README.md`のゲーム一覧に追加してください。

## 推奨される次のゲーム

以下は実装候補の有名なゲームです：

### クラシックゲーム
- **スネーク (Snake)** - シンプルな移動ゲーム
- **テトリス (Tetris)** - ブロック配置パズル
- **マインスイーパー (Minesweeper)** - 地雷探しゲーム
- **パックマン (Pac-Man)** - 迷路ゲーム
- **ブロック崩し (Breakout)** - ボール反射ゲーム
- **インベーダー (Space Invaders)** - シューティングゲーム

### ボードゲーム
- **チェス (Chess)** - 戦略ボードゲーム
- **オセロ (Reversi)** - 石返しゲーム
- **将棋 (Shogi)** - 日本の戦略ゲーム
- **四目並べ (Connect Four)** - 縦型三目並べ

### カードゲーム
- **神経衰弱 (Memory/Concentration)** - 記憶ゲーム
- **ソリティア (Solitaire)** - 一人遊び用カードゲーム
- **ブラックジャック (Blackjack)** - カジノゲーム

## コーディング標準

### Python
- PEP 8スタイルガイドに従う
- 型ヒントの使用を推奨（Python 3.6+）
- docstringで関数を文書化

### JavaScript
- ES6+の機能を使用
- セミコロンの使用
- JSDocでコメント

### 共通
- 意味のある変数名を使用
- コードの重複を避ける
- エッジケースを考慮
- エラーハンドリングを適切に実装

## テスト

各実装には基本的なテストを含めてください：
- 初期化テスト
- 基本的な操作テスト
- 勝利/敗北条件のテスト
- エッジケースのテスト

## 貢献方法

1. このリポジトリをフォーク
2. 新しいブランチを作成 (`git checkout -b feature/new-game`)
3. 変更をコミット (`git commit -am 'Add new game'`)
4. ブランチにプッシュ (`git push origin feature/new-game`)
5. プルリクエストを作成
